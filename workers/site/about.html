About MySweetPea

Self-hosted services for a privacy-first community.

Our Mission

I am someone who cares deeply about privacy. In the modern world where privacy is cheap and personal data is the product, I wanted to give something back to the community. MySweetPea is my way of providing secure, private, self-hosted alternatives to the services we all use every day — without the tracking, the ads, or the corporate overhead.

I also genuinely enjoy doing this. Building and maintaining this infrastructure is a passion project. Every service running here is one less reason to depend on a company that does not have your best interests at heart.

Why Self-Hosting?

I host everything locally on my own hardware. I do not rent cloud servers or use third-party hosting providers to run the services. The only external cost is a small VPS used as a relay for public access to media streaming.

This means:

  • Full control over the infrastructure and data
  • No third-party access to your data — not even mine, since services like Vaultwarden use zero-knowledge encryption
  • No vendor lock-in — if I shut down, you still have your data
  • Transparency — all infrastructure code is public on GitHub

The Cost of Running MySweetPea

Running these services is not free, which is why donations matter. The main expenses include:

  • Electricity — three servers running 24/7 consume a significant amount of power
  • Hardware — the physical servers, networking equipment, and storage drives
  • Internet bandwidth — serving media and services to multiple users requires a fast connection
  • Domain registration — the mysweetpea.cc domain needs annual renewal
  • VPS relay — a small virtual server for public access to media streaming (currently free tier)

Donations help offset these ongoing costs and fund future hardware upgrades and new services. Without community support, this project would not be sustainable.

Open Source

Every service we run is open source. Our infrastructure code is public on GitHub for full transparency. You can inspect exactly how everything is configured, deployed, and secured.
